  ★★★  最重要的一件事：貼在 LINE 裡的網址要加尾巴  ★★★

  以後你要在 LINE（社群、聊天室、群組）貼網站連結，
  請一律用這一條，不要用原本那條：


    https://wsad952701-beep.github.io/123/13/?openExternalBrowser=1
                                            ^^^^^^^^^^^^^^^^^^^^^^^
                                            這個尾巴不能少！


  為什麼？
    LINE 內建的瀏覽器沒辦法叫起 LINE App（等於叫自己），
    所以客人在 LINE 裡開網頁，點「加入社群」就會卡在
    「請啟動 LINE 應用程式」——就是你剛剛遇到的那個畫面。

    加上這個尾巴，LINE 就會改用手機的 Safari / Chrome 開，
    在那裡點社群連結才叫得起 LINE App，一切正常。

  ※ 網頁本身也做了自動偵測：萬一客人還是用舊網址進來，
    會自動幫他彈到瀏覽器。但你貼新網址最保險。


------------------------------------------------------------------
  你要做的：把 2 個檔案傳上 GitHub
------------------------------------------------------------------

   只要換  index.html  和  config.js  這 2 個檔。
   （assets 的 24 張照片你 GitHub 上本來就有，不用重傳）

   手機做法：
     1. 打開 github.com/wsad952701-beep/123
     2. 進 13 資料夾
     3. 右上角 → Add file → Upload files
     4. 把 index.html 和 config.js 丟進去（會自動覆蓋舊的）
     5. Commit changes
     6. 等 2 分鐘 → 完成！

   ※ 社群連結已經幫你填在 config.js 裡了，什麼都不用設定。

------------------------------------------------------------------
  萬一客人還是卡在 LINE 內建瀏覽器
------------------------------------------------------------------

   網頁底部會自動跳出一條綠色提示，教他：

     1. 點頁面右上角的「⋯」
     2. 選「用預設瀏覽器開啟」
     3. 就可以正常點餐了

   也有一顆「直接點我，用瀏覽器開啟」的按鈕。


------------------------------------------------------------------
  訂單為什麼要客人自己貼？
------------------------------------------------------------------

   LINE 官方規定：社群（OpenChat）不支援網頁直接發訊息。
   全世界的網站都做不到，不是我們的問題。

   所以最可靠的方式就是「自動複製 → 開社群 → 貼上」，約 10 秒。
   送出後畫面會有三步驟圖和綠色大按鈕，客人照做就好。

   點餐單不會提早清空，一定要客人按「我已經貼上了」才清，
   免得他還沒貼上，單子就不見了。


------------------------------------------------------------------
  想更進階？（選配，不做也沒關係）
------------------------------------------------------------------

   去 LINE 拿一組 LIFF ID 填進 config.js，
   客人加入社群後會再跳一次「LINE 官方授權畫面」，
   按「允許」才能點餐 —— 你就能拿到他真實的 LINE 名稱。

   ★ 不需要 LINE 官方帳號，用你自己平常的 LINE 帳號就好，免費。

   詳細步驟看 SETUP.md。
