## 怎麼上 GitHub Pages

1. 建一個新的 repository（**Public**，Private 要付費才能開 Pages）
2. 把這個資料夾裡的**所有東西**上傳到 repo 根目錄
   （`index.html` 一定要在最外層，不能包在子資料夾裡）
3. repo 頁面 → **Settings** → 左邊 **Pages**
4. Source 選 **Deploy from a branch**，Branch 選 `main` / 資料夾選 `/ (root)` → **Save**
5. 等 1～2 分鐘，網址會出現在同一頁：
   `https://你的帳號.github.io/倉庫名稱`

> `.nojekyll` 是隱藏檔（開頭有一點）。用網頁介面拖曳上傳時可能看不到它，
> 沒關係——這個網站沒有底線開頭的檔名，少了它也能正常運作。

## ★ 上線後要改的一件事：網址

`index.html` 裡面有 **4 處** 寫著 `https://www.shunji-roast.tw`。
用編輯器「全部取代」換成你的實際網址即可：

```
https://www.shunji-roast.tw   →   https://你的帳號.github.io/倉庫名稱
```

這 4 處分別是：

```html
<meta property="og:image" content="...">      <!-- 預覽圖 -->
<meta property="og:url"   content="...">      <!-- 分享網址 -->
<link  rel="canonical"    href="...">         <!-- SEO 正規網址 -->
"image": "..."                                <!-- JSON-LD 結構化資料 -->
```

4 處必須**完全一致**（有 `www` 就都要有）。

**不改也能正常瀏覽、正常點餐**，只是分享到 LINE 時不會跳出預覽縮圖。

之後若換成自訂網域（例如 `www.shunji-roast.tw`），
在 Settings → Pages → Custom domain 填入，再把這 4 處改回來就好。

## 線上點餐怎麼運作

客人在菜單按 `＋` → 打開「點餐單」填訂購資料 → 按「送出訂單」
→ 系統自動把整張訂單複製到剪貼簿 → 跳出 LINE 社群 QR
→ 客人加入社群後長按貼上、送出，訂單就到店家手上。

第二次以後系統會記得他加過社群，直接顯示「前往社群貼上訂單」。

訂單內容包含：訂購單位、聯絡人、電話、取餐方式（外送／自取）、
外送地址、送達時間、備註、餐點明細（含份數）、合計、訂單編號。

**份數顯示**：菜單上按 `＋` 之後按鈕會就地變成 `−  份數  ＋`，
點錯直接按 `−` 就能減少或取消，減到 0 自動移除。
點餐單保留 12 小時，重新整理或切去 LINE 再回來都不會不見。

---

## 沒有後端

訂單不會存到伺服器，是靠「複製到剪貼簿 → 貼到 LINE 社群」完成的。
所以不需要資料庫、不需要主機、也不用擔心個資存在雲端。

## 瀏覽器支援

Chrome / Safari / Edge / Firefox 近兩年版本，以及 LINE 內建瀏覽器。
無法使用剪貼簿 API 的環境會自動改用備援方式；真的複製不了時，
彈窗會提示「展開下方手動複製」。
