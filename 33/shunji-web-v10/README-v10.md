# 順記燒臘館 v10 升級說明

## 🆕 v10 新功能總覽

### 1. 📊 儀表板（Dashboard）── 全新頁面
一進後台就看到今日經營狀況一目了然：
- **今日總份數** ── 所有品項今日加起來共幾份
- **已售出** ── 今天已經賣了幾份
- **剩餘** ── 還可以賣幾份
- **售完品項數** ── 今天有幾樣賣光了
- **⚠️ 低庫存警示** ── 剩 5 份以下的品項自動列出
- **⭐ 今日特餐** ── 今日特餐的預覽（價格、數量）
- **📋 最近訂單** ── 從這台裝置送出的最新 3 筆訂單

### 2. ⭐ 今日特餐（大幅強化）
每個特餐品項現在有 **三個** 可調整的參數：
| 品項 | 特價 | 限量份數 | ✕ |
|------|------|----------|---|
| ⭐ 順記三寶飯 | 99 | 20 | 移除 |

- **特價**：留空 = 用原價／後台改過的價格；有填數字 = 特惠價（前台會劃掉原價）
- **限量份數**：留空 = 不限量；有填數字 = 限量供應，賣完自動下架
- 限量份數會**自動同步**到「庫存管理」的今日份數

### 3. 📦 庫存管理（原「數量管理」改名 + 增強）
每個品項現在有更清楚的狀態顯示：
- 🔵 **今日 N 份** ── 你設定的今日總份數
- 🟢 **已售 N** ── 今天已經賣了多少
- 🟡 **剩 N** ── 還剩多少可以賣
- 🔴 **售完** ── 全部賣光了

新增功能：
- **± 快速調整**：− 是「賣出 1 份」（已售 +1），＋ 是「補貨 1 份」（總數 +1）
- **批次操作**：一鍵全部設 20/30/50 份、一鍵重置今日已售、一鍵清空所有設定
- **搜尋框**：快速找到品項
- **顏色警示**：售完的一列變紅，剩 5 份以下變黃

### 4. 📋 訂單記錄 ── 全新頁面
客人送出訂單後會**自動記錄**在後台：
- 顯示：訂單時間、品項、每項數量、單價、小計、總金額
- 狀態：⏳ 待確認 / ✓ 已扣庫存
- 操作：
  - **✓ 確認扣庫存** ── 一鍵把訂單品項的數量從「今日份數」扣掉
  - **↩ 取消扣庫存** ── 誤按了？可以還原
  - **刪除紀錄** ── 只刪紀錄，不影響已扣的庫存
  - **批次確認** ── 一鍵處理所有待確認的訂單

⚠️ **重要**：訂單記錄只在**客人下單的那台裝置**上有。所以你（店家）在自己的裝置看不到客人的訂單記錄 —— 這是刻意的設計，因為訂單本來就是透過 LINE 社群送給你。你在 LINE 收到訂單後，可以直接在「庫存管理」用 − 按鈕手動扣，或請客人告訴你他們裝置上有幾筆訂單。

### 5. 🔥 前台即時庫存回饋
- 客人加餐點到點餐單時，「**剩 N 份**」的標籤**即時減少**
- 讓客人看到「還剩多少可以搶」，製造適度的搶購感
- 只是這台手機的視覺效果，不會影響其他客人看到的庫存

### 6. 🎨 後台專業改版
- **新的視覺設計**：仿 Notion / Linear 的儀表板風格
- **統計卡片**：漂亮的四個大數字卡片
- **顏色警示**：低庫存黃色、售完紅色、正常綠色
- **搜尋框**：菜單管理、庫存管理都可以搜尋
- **批次操作**：省下一個個調的時間
- **順滑動畫**：切換頁籤有平滑過渡

### 7. 🔒 自動售完偵測
- 只要品項的「剩餘 = 0」，就自動視為售完
- 客人前台看到「售完」標籤、按鈕變灰、不能點餐
- 不用手動去點 🚫 售完按鈕（除非你要提前手動標）

## 🚀 怎麼用

### 方法 A：一鍵發佈（推薦）

1. 打開 `admin.html` → 登入
2. 到「發佈設定」頁籤 → 填入 GitHub 資料：
   - **Owner**：你的 GitHub 帳號
   - **Repo**：你的倉庫名稱
   - **Path**：`menu-data.js` 的路徑（例：`13/menu-data.js` 或 `menu-data.js`）
   - **Token**：到 github.com/settings/tokens → Generate → 勾 repo → 複製
3. 按「儲存設定」（只要設一次）
4. 以後每次改完 → 按底部「📡 一鍵發佈給所有客人」就搞定

### 方法 B：手動發佈

1. 在 `admin.html` 改完
2. 到「發佈設定」→ 「手動發佈」→ 複製設定碼
3. 到 GitHub 打開 `menu-data.js` → 全選貼上 → Commit

## 📋 每日流程建議

### 開店前（早上）
1. 打開後台 → 儀表板看昨天狀況
2. 到「庫存管理」→ 「重置今日已售」按鈕（清零昨天的銷售）
3. 設定今日份數（可用批次操作「全部設 30 份」快速搞定）
4. 如有特餐 → 「今日特餐」設定特價與限量
5. 底部「📡 一鍵發佈」

### 營業中
- 收到 LINE 訂單 → 打開後台 → 「庫存管理」→ 賣出的品項按 − 一次或多次
- 或者：客人如果也是用同一台裝置測試 → 直接到「訂單記錄」按「✓ 確認扣庫存」
- 補貨？→ 「庫存管理」按 ＋
- 有品項不賣了？→ 「菜單管理」按 🚫 售完

### 打烊後
- 「儀表板」看今天賣多少
- 儀表板 → 記錄好今日銷售

## ⚠️ 注意事項

### 訂單記錄的限制
- 訂單記錄存在**客人自己的裝置**上（localStorage）
- 你（店家）在自己的裝置**看不到**其他人的訂單
- 這是設計上的限制（沒有真正的後端伺服器）
- 訂單本來就會送到 LINE 社群 → 你在 LINE 看訂單、在後台手動扣庫存就好

### 庫存扣減的兩種方式
1. **手動扣**（推薦，最可靠）：LINE 收到訂單 → 到「庫存管理」按 − 幾次
2. **自動扣**（同一台裝置）：客人在你的裝置測試訂餐 → 「訂單記錄」按「✓ 確認扣庫存」

### 每天要記得
- 開店前**重置今日已售**（「庫存管理」→ 「重置今日已售」按鈕）
- 不然「已售」會一直累加，剩餘會看起來很少

### GitHub Token 存哪裡
- 存在你自己手機／電腦的瀏覽器裡（localStorage）
- 不會被其他人看到
- 如果換裝置了 → 重新到「發佈設定」填一次就好

### 圖片大小限制
- 建議 1MB 以內
- 系統會自動幫你縮到 1200px
- 太大存不進 localStorage → 換張小一點的

## 🐛 疑難排解

**Q: 後台改了東西，客人為什麼沒看到？**
A: 你只按了「先存本機」，那只有你這台看得到。要按「📡 一鍵發佈」才會給所有客人。

**Q: 一鍵發佈按下去失敗？**
A: 檢查「發佈設定」的 GitHub 資料是否正確（Owner、Repo、Path、Token）。Token 是否過期。

**Q: 客人看到的是舊的？**
A: GitHub Pages 有 2 分鐘快取。等等就好。

**Q: 帳密忘了？**
A: 打開 `admin.js` 最上面兩行看。或修改後上傳新的。

**Q: 打開後台空白？**
A: 檢查 `menu-data.js` 和 `admin.js` 都在同一個資料夾。用瀏覽器開發者工具看有沒有錯誤。
