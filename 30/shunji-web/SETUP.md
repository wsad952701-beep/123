# 一、★ 訂單圖：9:16 直式

## 為什麼要改

LINE 的圖片預覽是**直式**的。

舊版訂單圖的高度是「跟著內容跑」的：

| 客人點幾樣 | 舊版圖片 | 在 LINE 裡看起來 |
|---|---|---|
| 2 樣 | 矮矮一張 | 還可以 |
| 12 樣 | 長長一條 | **被壓成小小一格，要點開放大才看得到字** |

現在**固定 1080 × 1920（9:16）**，不管點幾樣都一樣：

- **點得少** → 空下來的地方自動變成「店章 + 提醒」，看起來像一張正式訂單聯
- **點得多** → 中間整塊自動等比縮小，剛好塞滿，**不會裁到字、不會爆版**
- **點到二十幾樣** → 尾巴收成一行「…另 N 項，詳見文字訂單」
  （合計金額**還是全部算進去**，不會少算）

> 已經算過：從「點 1 樣」到「整份菜單 30 項全點」都塞得下。
> 一般公司訂便當（10 項以內）字級維持最大。

## 字級：先「盡量放大」，塞不下才慢慢縮

**這是新的邏輯。** 舊版字級寫死，短單留一大片空白卻不放大字。
現在改成：**字先撐大到 1.5 倍，撐不下才開始縮。**

| 點幾樣 | 品項字級 | 說明 |
|---|---|---|
| 1–5 樣 | **44px** | 吃滿 1.5 倍，字最大 |
| 8 樣 | 36px | 開始縮 |
| 12 樣 | 28px | |
| 20 樣 | 20px | |
| 30 樣（全點） | 14px | 縮到底，但一項都不會漏 |

一般公司訂便當（5 樣以內）字最大。

### 一張 3 項的單，舊 vs 新

| 位置 | 舊 | 新 | |
|---|---|---|---|
| 品項名稱 | 28px | **44px** | +57% |
| 單價 × 數量 | 24px | **41px** | +71% |
| 小計金額 | 26px | **50px** | +92% |
| 地址・時間 | 33px | **51px** | +55% |
| 幾項幾件 | 17.5px | **24px** | +37% |
| 頁尾三行 | 21/17.5/15.5px | **24/20/18px** | +14~16% |

## ★ 品項列多了「單價」

舊版只有「×2 份」和小計，**看不到單價**，沒辦法核對。現在是：

```
品　項                單價 × 數量      小計
碳燒原隻燒鵝腿飯        220 × 2 份       440
碳燒原隻燒鴨腿飯        160 × 2 份       320
順記四寶飯             150 × 2 份       300
```

一眼就能核對「單價對不對、有沒有算錯」。文字訂單也一起加了。

> 欄寬是**量出來的**，不是寫死的 —— 品名絕對不會被切掉。
> 萬一最長的品名（碳燒脆皮大鵝（1/4隻））真的擠不下，
> 會把品名字級調小 1–2px，而不是切字。

網站上的字也調大了 —— **越小的字加得越多**：最小的 9.5px → 11.5px（+21%），
一般內文 15px → 16.6px（+11%），大標題只加 3%。

> 順便修好一個原本就有的 bug：螢幕寬 521–660px（小平板、手機橫拿）時，
> 導覽列的「店名 + 用戶徽章 + 電話號碼」會擠爆、跑出畫面外。
> 已測 27 種螢幕寬度 × 3 種登入狀態，全部正常。

## 三種裝置，自動走不同的路（都不會卡住）

| 客人用什麼開網頁 | 按下綠色鈕會發生什麼 |
|---|---|
| **手機瀏覽器**（Safari / Chrome）★最順 | 叫出系統分享 → 選 LINE → 選社群 → 傳送。**傳的是圖片** |
| **LINE 內建瀏覽器**（從 LINE 點連結進來的） | LINE 內建瀏覽器多半不給網頁「一鍵分享圖片檔」→ 綠色鈕自動變成**兩步**：<br>① 存訂單圖片 → 存進相簿<br>② 開啟社群 → 按「＋」→ 相簿 → 選訂單圖 → 傳送<br>畫面也會教他：右上「⋯」→ 用預設瀏覽器開啟，就變一鍵傳圖。<br>趕時間還有【改傳文字訂單】可以按 |

## 為什麼最後那一下「傳送」還是要客人自己按？

這是防垃圾訊息的安全機制，**全世界的網站都做不到**，不是我們的問題。
LINE 也沒有開放任何 API 讓網頁「自己」把訊息或圖片丟進社群（OpenChat）。

我們能做的都做完了 —— 圖片幫他畫好、對象幫他標好，只剩最後那一下【傳送】。

### 💡 換照片的建議做法

在後台換照片，照片會被轉成一大串文字塞進 `menu-data.js`，**檔案會變很大**。

**只是要換照片的話，比較省的做法是**：
直接把新照片上傳到 GitHub 的 `assets/` 資料夾，**覆蓋同名的舊檔**
（例如把新的三寶飯照片存成 `dish_sanbao.webp` 上傳覆蓋）。
這樣網頁會自動用新照片，`menu-data.js` 完全不用動。

後台換照片的功能，適合**臨時**想試效果、或**趕時間**的時候用。

# 出問題怎麼辦

| 狀況 | 原因 | 怎麼修 |
|---|---|---|
| 進不了後台 | 密碼打錯 | 檢查 `admin.js` 最上面兩行 |
| 改了價格但客人看到舊的 | **忘記發佈** | 照上面「發佈給客人」做一遍 |
| 換照片說「太大」 | 照片超過 1MB | 用手機「編輯 → 裁切」縮小再試 |
| 儲存失敗 | 存太多張照片 | 按「全部復原」，或改用上傳 assets 的方式 |
| 按送出沒跳去 LINE | 瀏覽器擋彈窗 | 會自動改成直接跳轉，再試一次 |
| 手機沒裝 LINE | — | 會跳到 LINE 下載頁 |
| 訂單圖上字變小了 | 客人點了很多樣 | **正常**。為了維持 9:16 會自動縮，塞得下就好 |
| 改了檔案但沒變化 | GitHub / 瀏覽器快取 | 等 2 分鐘，關掉分頁重開 |

# 其他設定

### 改營業時間
`index.html` 搜尋 `var HOURS`。數字是「從午夜起算的分鐘數」：
```js
1: [[660,810],[960,1140]],   // 週一 11:00–13:30、16:00–19:00
6: [],                       // 週六公休
```
※ 改了記得**同時**改「門市」區塊那張表，還有 JSON-LD 裡的 `openingHoursSpecification`。

### 換社群連結
`config.js` 的 `COMMUNITY_URL`。

### 訂單要不要做成圖片
`config.js` 的 `ORDER_IMAGE`（`true` = 傳圖，`false` = 只傳文字）。

### 讓客人先挑菜、送出時才要加入社群
`index.html` 搜尋 `LOGIN_BEFORE_ADD`，改成 `false`。

### 拿到客人真實的 LINE 名稱（選配）
去 `https://developers.line.biz/console/` 申請一組 LIFF ID，
填進 `config.js` 的 `LIFF_ID`。不需要官方帳號，用自己的 LINE 帳號就好，免費。
