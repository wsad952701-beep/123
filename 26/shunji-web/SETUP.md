# 一、店家後台

## 怎麼進去

| 方法 | 做法 |
|---|---|
| **A** | 網址後面加 `#admin`<br>`https://wsad952701-beep.github.io/123/13/#admin` |
| **B** | 連續快點網頁最下面的**店名 5 下** |

輸入帳密 → 進去了。登入狀態保留 **8 小時**。

## 能改什麼

### 【價格・售完・下架】

- **價格** — 直接改數字，改過的會變綠色，下面會顯示「原 140 → 150」
- **🚫 售完** — 客人**看得到但點不了**（變灰色 + 打上「售完」標籤）
- **👁 下架** — 整個**藏起來**，客人完全看不到

> 已經在客人點餐單裡的品項，如果被標成售完或下架，會自動被移除，
> 免得客人結帳時才發現不能賣。

### 【換照片】

按「換照片」→ 從相簿選一張 → 馬上換好。

- 系統會**自動縮到 1200px、自動壓縮**
- 太大（超過 1MB）會擋下來，請先用手機的「編輯 → 裁切」縮小

### 改完一定要按

底下的 **【儲存並套用】** → 網頁重新整理 → 新的就出來了。

想反悔？按 **【全部復原】** 回到原本的樣子。

---

## ★★★ 最重要：發佈給客人 ★★★

> ### ⚠️ 在後台改完，**只有你這支手機**看得到！
> ### 客人打開網站，看到的還是**舊的**！

要讓**所有客人**都更新，一定要「發佈」：

| 步驟 | 做什麼 |
|---|---|
| 1 | 後台 → 切到 **【發佈給客人】** 頁籤 |
| 2 | 按 **【📋 複製設定碼】** |
| 3 | 回 GitHub → 進 `13` 資料夾 → 點開 **`menu-data.js`** |
| 4 | 點右上角鉛筆 ✏️ |
| 5 | 裡面**全部刪光** → **貼上**剛剛複製的 |
| 6 | **Commit changes** |
| 7 | 等 2 分鐘 → **全部客人都更新了** ✓ |

> 懶得複製貼上？按 **【⬇ 下載 menu-data.js】**，
> 直接把下載到的檔案上傳覆蓋 GitHub 上的同名檔案，效果一樣。

### 💡 換照片的建議做法

在後台換照片，照片會被轉成一大串文字塞進 `menu-data.js`，**檔案會變很大**。

**只是要換照片的話，比較省的做法是**：
直接把新照片上傳到 GitHub 的 `assets/` 資料夾，**覆蓋同名的舊檔**
（例如把新的三寶飯照片存成 `dish_sanbao.webp` 上傳覆蓋）。
這樣網頁會自動用新照片，`menu-data.js` 完全不用動。

後台換照片的功能，適合**臨時**想試效果、或**趕時間**的時候用。

# 二、一鍵送出訂單

## 客人現在的流程

```
   點好餐 → 填資料 → 按【送出訂單】
        │
        ▼
   ★ 手機直接跳進 LINE
     「傳送給…」畫面
     訂單已經幫他打好在裡面了
        │
        ▼
   選【順記社群】→ 按【傳送】 → 完成 ✓
```

**不用複製、不用貼上、不用切來切去。**

## 用的是什麼技術

LINE 官方的 URL Scheme：

```
https://line.me/R/share?text=<訂單內容>
```

這是 LINE 官方文件裡寫的正式用法，會打開「傳送給…」畫面，
讓使用者**選聊天室**，而**訊息內容已經預先填好**。

## 為什麼最後那一下「傳送」還是要客人自己按？

> **LINE 官方規定：網頁不可以自動幫使用者按下傳送鍵。**

這是防止垃圾訊息的安全機制，**全世界的網站都做不到**，不是我們的問題。

但比起以前已經好非常多：

| | 以前 | 現在 |
|---|---|---|
| 步驟 | 複製 → 開 LINE → 長按輸入框 → 貼上 → 傳送（**5 步**） | 按按鈕 → 選聊天室 → 傳送（**3 步**） |
| 會貼錯嗎 | 會（複製失敗、貼到別的地方） | 不會（內容是系統填的） |
| 客人要學嗎 | 要看三步驟圖 | 不用，跟著跳就對了 |

## 在 LINE App 裡開網頁的話

如果客人是**從 LINE 的聊天室點連結**進來的（一對一 / 群組 / 多人聊天室），
而且你有填 `LIFF_ID` —— 那訂單會**直接送進那個聊天室**，
連「選聊天室」那步都不用，**一步都不用做**。

（社群 OpenChat 例外，LINE 不支援，會走上面的一鍵送出）

# 出問題怎麼辦

| 狀況 | 原因 | 怎麼修 |
|---|---|---|
| 進不了後台 | 密碼打錯 | 檢查 `admin.js` 最上面兩行 |
| 改了價格但客人看到舊的 | **忘記發佈** | 照上面「發佈給客人」做一遍 |
| 換照片說「太大」 | 照片超過 1MB | 用手機「編輯 → 裁切」縮小再試 |
| 儲存失敗 | 存太多張照片 | 按「全部復原」，或改用上傳 assets 的方式 |
| 按送出沒跳去 LINE | 瀏覽器擋彈窗 | 會自動改成直接跳轉，再試一次 |
| 手機沒裝 LINE | — | 會跳到 LINE 下載頁 |
| 改了檔案但沒變化 | GitHub / 瀏覽器快取 | 等 2 分鐘，關掉分頁重開 |

# 其他設定（沒變）

### 改營業時間
`index.html` 搜尋 `var HOURS`。數字是「從午夜起算的分鐘數」：
```js
1: [[660,810],[960,1140]],   // 週一 11:00–13:30、16:00–19:00
6: [],                       // 週六公休
```

### 換社群連結
`config.js` 的 `COMMUNITY_URL`。

### 讓客人先挑菜、送出時才要加入社群
`index.html` 搜尋 `LOGIN_BEFORE_ADD`，改成 `false`。

### 拿到客人真實的 LINE 名稱（選配）
去 `https://developers.line.biz/console/` 申請一組 LIFF ID，
填進 `config.js` 的 `LIFF_ID`。不需要官方帳號，用自己的 LINE 帳號就好，免費。
